# raylib

Koda bindings for **raylib 5.x** (generated by wrapgen). The **official Koda SDK** ships this folder, the C glue, and **vendored raylib** under `third_party/raylib_static/stage/` so you can build games without installing LLVM or raylib separately (static link uses the bundled `libraylib.a`; Windows also includes `raylib.dll` in `lib/` for dynamic setups).

**Full guide:** [docs/guides/raylib.md](../../docs/guides/raylib.md) (include paths, `KODA_NATIVE_SOURCES`, DLLs, Linux ARM64).

## Quick start

**1.** Include the bindings (path is resolved from the SDK‚Äôs `wrappers/` tree ‚Äî see the guide):

```koda
#include "raylib/raylib.koda"
```

**2.** Point native build at the glue file:

```powershell
set KODA_NATIVE_SOURCES=wrappers\raylib\wrapper.c
```

```bash
export KODA_NATIVE_SOURCES=wrappers/raylib/wrapper.c
```

**3.** Build from the SDK root (or set `KODA_RAYLIB_STAGE` / work under a layout that still finds `wrappers/` and `third_party/`):

```powershell
koda build mygame.koda -o mygame.exe
```

If the shipped **`third_party/raylib_static/stage`** tree is present, Koda adds include + static library flags automatically (`KODA_USE_VENDORED_RAYLIB`, default on).

**4.** **Windows:** If your executable needs **`raylib.dll`**, copy `third_party/raylib_static/stage/lib/raylib.dll` next to the `.exe` or add it to **`KODA_BUNDLE_FILES`** for `koda bundle`.

**5.** **Manual link** (no vendored tree, or extra libs): set **`KODA_LINKFLAGS`** to your `-I`, `-L`, and `-lraylib` (and OS frameworks on macOS). See [docs/wrappers.md](../../docs/wrappers.md).

## Troubleshooting

| Issue | What to check |
|--------|----------------|
| Undefined symbol in `wrapper.c` | `KODA_NATIVE_SOURCES` includes `wrapper.c`; `KODA_LINKFLAGS` / vendored `stage/` finds raylib. |
| Could not find `raylib/raylib.koda` | Run with SDK `koda` so `wrappers/` is next to the compiler, or set **`KODA_PATH`** to the SDK root. |
| Crash on start (Windows) | Missing **`raylib.dll`** next to the exe ‚Äî copy from `stage/lib/`. |
